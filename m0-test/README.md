# M0 技術驗證 — 操作步驟

這是丟棄式的迷你測試擴充功能，只做一件事：確認擴充功能發出的請求能不能讓微博認得你的登入狀態。測完、把結果回報後，整個 `m0-test/` 資料夾就可以刪掉。

## 步驟

1. **登入微博**
   在 Chrome 開 https://weibo.com 和 https://m.weibo.cn 各看一眼，確認兩邊都是登入狀態（看得到你的帳號頭像）。

2. **載入這個測試擴充功能**
   1. 在網址列輸入 `chrome://extensions` 按 Enter
   2. 打開右上角的「開發人員模式」開關
   3. 點左上角「載入未封裝項目」
   4. 選擇這個資料夾：`桌面\weibo-chaohua-extension\m0-test`
   5. 列表中出現「微博超話簽到 M0 技術驗證」即成功

3. **執行測試**
   1. 點瀏覽器右上角的拼圖圖示 🧩，找到「微博超話簽到 M0 技術驗證」點開（可順手釘選）
   2. 點「▶ 全部執行」，等它依序跑完 4 項測試（約半分鐘）
   3. 點「複製全部結果」

4. **把結果貼回給 Claude**
   直接把剪貼簿內容貼到對話裡即可。

## 注意事項

- 測試 3、4 可能會自動開啟微博的分頁，這是正常的（它需要借那個頁面發請求）
- 測試過程中會對「一個」超話真的簽到一次（只有一個，不會批量掃）
- 如果視窗不小心關掉了，重新點開擴充功能圖示、再按一次「全部執行」即可
- 全部顯示 ❌ 也是有價值的結果，照樣貼回來

## 測試項目對照

| 按鈕 | 驗證的組合 |
| --- | --- |
| 測試 1 | 手機版 API（m.weibo.cn）× service worker 直接 fetch |
| 測試 2 | 電腦版 API（weibo.com）× service worker 直接 fetch |
| 測試 3 | 手機版 API × 注入微博頁面執行 |
| 測試 4 | 電腦版 API × 注入微博頁面執行 |

只要任何一項全綠（登入偵測、列表、簽到都 ✅），專案就確定可行。
